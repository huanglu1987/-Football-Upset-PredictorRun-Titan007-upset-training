# 外用制剂临床开发策略 SKILL 设计文档

## 1. 背景

目标是设计一个面向皮肤外用、局部起效的小分子制剂的临床开发策略 SKILL。用户输入具体品种信息后，SKILL 输出一套结构化的临床开发路径建议，覆盖中国和 FDA 两套监管场景，并显式比较保守路径与激进路径。

该 SKILL 的目标不是直接替代最终版 protocol 撰写，也不是替代法规事务或统计学的正式判断，而是作为上游的临床开发策略设计助手，帮助用户快速形成一套可解释、可追溯、可进一步细化的研发路径。

## 2. 产品目标

### 2.1 核心目标

- 输入单个目标品种的基础信息后，输出从早期研究到确证性研究的完整开发路径。
- 同时支持中国、美国和中美双报三种场景。
- 同时覆盖创新药与改良型新药，第一版优先把改良型新药规则做细。
- 每条关键建议都附带依据说明，而不是只输出结论。
- 针对同一品种默认输出两套方案：
  - 保守路径
  - 激进路径

### 2.2 非目标

- 不直接生成完整 protocol 正文。
- 不替代样本量正式计算。
- 不替代最终注册分类结论。
- 不把单一 FDA 个案直接推广为普适监管要求。

## 3. 第一版范围

### 3.1 剂型范围

第一版覆盖以下皮肤外用、局部起效小分子剂型：

- 乳膏
- 凝胶
- 软膏
- 乳液
- 搽剂
- 喷雾
- 泡沫
- 涂膜剂

不考虑贴剂。

### 3.2 注册路径范围

- 创新药
- 改良型新药

其中第一版优先把改良型新药相关规则做细，尤其是以下场景：

- 口服改外用
- 已知活性成分改良剂型或处方工艺
- 目标是提高依从性、降低刺激性、降低系统暴露或改善局部安全性

### 3.3 优先适应症

第一版全部支持以下适应症进入总框架，但优先做深以下 4 个适应症模块：

- 痤疮
- 玫瑰痤疮
- 浅表真菌感染
- AGA

后续扩展适应症包括：

- AD
- 脂溢性皮炎
- 银屑病
- 斑秃
- 白癜风
- 放射性皮炎
- 汗孔角化症

## 4. 资料层设计

SKILL 采用分层资料架构，避免把不同证据等级混为一谈。

### 4.1 第一层：核心规则层

优先级最高，直接写入规则树。

中国侧核心资料至少包括：

- 《局部给药局部起效的药物临床试验技术指导原则》
- 《化学药品改良型新药临床试验指导原则》
- 《化学药改良型新药临床药理学研究技术指导原则（试行）》
- 《药物临床试验申请临床评价技术指导原则》
- 《药物暴露-效应关系研究技术指导原则》

美国侧核心资料至少包括：

- FDA 505(b)(2) 路径总体指导原则
- FDA 关于外用产品最大使用研究的指导原则
- 与目标适应症直接相关的正式 guidance for industry

### 4.2 第二层：审评实务层

用于补充“监管机构实际如何接受某类路径”的信息。

- FDA 公开审评资料
- FDA review package
- FDA multidisciplinary review
- 产品标签与审批文件
- 用户自建的 `505(b)(2)` 外用制剂案例汇总

该层资料用于解释审评倾向、桥接接受度、长期安全要求、关键终点处理方式，但不能直接压过正式指导原则。

### 4.3 第三层：辅助技术层

该层主要服务复杂外用制剂的技术判断。

- FDA PSG
- 外用制剂相关体外释放、体外透皮、Q1/Q2/Q3、局部 BE 思路
- 与局部安全、光安全、刺激/致敏相关的方法学资料

PSG 主要是辅助规则，不直接作为创新药或 505(b)(2) 的硬要求，但对复杂剂型、等效性思路和监管偏好具有较高参考价值。

### 4.4 第四层：运行时检索层

当出现以下场景时，SKILL 必须运行时联网检索最新官方资料：

- 用户指定 FDA 路径
- 用户询问“最新”“目前”“现在”的要求
- 涉及某个具体品种、具体 PSG 或具体公开 review
- 涉及近两年可能新增或更新的 CDE/FDA 文件
- 输出中需要给出来源链接

## 5. 输入模型

输入模型分为必填、强烈建议填写、可选增强三层。

### 5.1 必填字段

- 分子或活性成分
- 剂型
- 目标浓度或规格
- 目标适应症
- 目标人群
- 开发地区：中国 / 美国 / 中美双报
- 注册路径初判：创新药 / 改良型新药 / 待判断
- 已有前期证据概况

### 5.2 强烈建议填写字段

- 活性成分既往上市基础
- 拟定临床优势
- 给药频次与疗程设想
- CMC 或制剂关键信息
- 主要监管担忧
- 计划对照方式

### 5.3 可选增强字段

- 非临床细节
- 早期临床细节
- 竞品和同类品种
- 商业和时间约束

### 5.4 输入不足时的处理

- 如果只有必填字段，输出初版开发策略，并标记高不确定项。
- 如果补齐强烈建议字段，输出正式版开发策略。
- 如果可选增强字段也较充分，则输出更高置信度的保守/激进双路径。

特别说明：目标浓度或规格缺乏既往外用依据，不作为硬停止条件。对于口服改外用或系统给药改局部给药项目，这是 SKILL 需要重点判断的典型场景。

## 6. 决策树主干

SKILL 主干按以下 8 步判断，每一步都输出判断结论、依据说明和对后续开发的影响。

### 6.1 监管场景判断

- 中国
- 美国
- 中美双报

### 6.2 注册属性判断

- 创新药
- 改良型新药
- 待判断

若为改良型新药，再细分：

- 同成分新剂型或新处方工艺
- 同成分新适应症
- 同成分新给药频次或依从性优化
- 同成分局部安全或系统暴露改善

### 6.3 既有知识基础判断

- 是否已有同成分口服上市
- 是否已有同成分外用上市
- 是否已有同适应症标准治疗
- 是否已有可引用的 FDA 公开审评先例

### 6.4 系统暴露风险判断

- 预期几乎无系统暴露
- 存在低水平系统暴露，但可控
- 可能存在不可忽略系统暴露

该节点直接影响：

- 是否需要最大使用 PK 或 MUsT
- 是否需要相对 BA 或桥接 PK
- 是否需要更强的暴露-效应分析
- 是否需要长期安全或系统风险强化评估

### 6.5 制剂复杂度与局部递送变化判断

- 常规半固体
- 喷雾 / 泡沫 / 涂膜剂
- 明显改变局部递送或释放特征的新型递送系统

若制剂学变化可能显著改变局部递送或全身吸收，系统会自动提高桥接不确定性等级。

### 6.6 探索性开发需求判断

- 是否需要完整 P1
- 是否可用局部安全 + 最大使用 PK 替代部分传统 P1
- 是否必须做剂量探索 P2
- 是否可合并部分探索和验证工作

### 6.7 确证性研究框架判断

- 单个关键研究还是两个关键研究
- 载体对照、阳性对照还是标准治疗对照
- 是否需要长期安全延伸
- 是否需要儿童、重症或特殊部位亚组

### 6.8 双路径生成

- 保守路径
- 激进路径

系统同时给出当前推荐路径，并解释推荐理由。

## 7. 双路径设计

### 7.1 保守路径

适用于以下情形：

- 制剂变化较大
- 外用浓度设定依据不足
- 局部递送或系统暴露判断不稳
- 儿童或特殊人群开发较早
- 计划中美双报且需兼顾较低监管风险

典型特征：

- 更完整的剂量探索
- 更充分的局部安全包
- 更稳健的桥接与长期安全策略
- 更少依赖单一案例推断

### 7.2 激进路径

适用于以下情形：

- 已有同成分或强同类经验
- 可引用较强的桥接基础
- 系统暴露较低且风险可控
- 用户明确接受更高沟通风险以换取更短周期

典型特征：

- 更强依赖桥接、暴露-效应和公开审评经验
- 尽量压缩早期开发或合并研究阶段
- 输出中必须显式标注监管沟通风险点和失败触发点

## 8. 输出模板

每次正式输出固定为 8 个模块。

### 8.1 项目结论摘要

- 产品定位
- 地区与注册路径判断
- 一句话总结推荐开发策略
- 一句话总结最大监管风险

### 8.2 开发前提判断

- 创新药或改良型新药的判断理由
- 是否存在可桥接基础
- 是否存在系统暴露关注
- 当前最影响路径选择的前提条件

### 8.3 双路径总览

- 保守路径摘要
- 激进路径摘要
- 推荐路径及推荐原因

### 8.4 分期开发策略

按 P1 / P2 / P3 或“早期 / 探索 / 确证”输出：

- 研究目的
- 推荐研究类型
- 推荐受试人群
- 推荐对照方式
- 主要终点与关键次要终点
- 样本量思路
- 是否可跳过或合并
- 进入下一阶段的决策门槛

### 8.5 关键专项研究建议

- 最大使用 PK / MUsT
- 局部刺激 / 致敏 / 光安全
- 暴露-效应分析
- 长期安全扩展
- 桥接研究
- 成分贡献研究或剂量探索

### 8.6 CDE vs FDA 差异提示

至少覆盖以下维度：

- 注册属性判断
- 临床优势要求
- 桥接受受度
- 系统暴露研究要求
- 长期安全要求
- 终点与证据强度
- 沟通建议

### 8.7 当前证据缺口清单

按优先级分层：

- 必须补
- 强烈建议补
- 可选增强

每条缺口后都标明影响哪一步决策。

### 8.8 依据说明

每条关键建议后标注：

- 依据类型：CDE 指导原则 / FDA 指导原则 / FDA 审评案例 / PSG / 一般研发逻辑
- 依据要点
- 适用前提

## 9. 四个优先适应症模块

所有适应症模块共用统一外壳，每个模块固定回答以下问题：

- 监管定位与常见开发路径
- 常见主要终点和关键次要终点
- 推荐疗程和关键评估时间点
- 早期研究需要优先解决的问题
- 确证性研究常见框架
- 保守/激进路径切换条件
- 优先调用的资料层

### 9.1 痤疮模块

重点包括：

- 炎症性皮损计数
- 非炎症性皮损计数
- IGA 或 EGSS 成功率
- 复方产品的成分贡献研究
- 局部刺激和长期耐受
- 青少年人群考虑

资料优先级高的来源包括：

- FDA 痤疮指导原则
- 痤疮相关公开审评资料
- 痤疮 505(b)(2) 案例

### 9.2 玫瑰痤疮模块

第一版聚焦丘疹脓疱型 rosacea。

重点包括：

- 炎症皮损绝对变化
- IGA 成功率
- 长期安全
- 面部敏感性和局部刺激
- 老年患者占比与分层

资料优先级高的来源包括：

- 玫瑰痤疮公开审评资料
- Zilxi、Epsolay 等典型案例

### 9.3 浅表真菌感染模块

内部优先细分：

- tinea pedis
- tinea corporis / cruris

重点包括：

- 真菌学终点
- 临床终点
- 综合治愈终点
- 治疗结束与随访时间点
- 复发与阴转不一致解释

该模块中 PSG 权重提高，但仍为辅助规则层。

### 9.4 AGA 模块

该模块为高价值但相对高不确定性模块。

重点包括：

- 目标区域毛发计数
- 标准化摄影或 phototrichogram
- 研究者总体评估
- 患者自评
- 暴露-效应分析
- 系统暴露、转移暴露和长期安全

如果涉及 topical finasteride 或类似路径，需显式强化：

- 系统暴露风险
- 女性接触与生殖安全沟通
- 风险可接受性判断

## 10. 失败保护与硬停止节点

SKILL 不应因信息不全而过早停止，但在以下情形下应给出强提醒或限制结论强度：

### 10.1 非阻断型风险提示

以下情况可以继续输出，但必须标记不确定性：

- 目标浓度或规格缺乏既往外用依据
- 早期数据不完整
- 仅有口服经验、缺少外用同类经验

### 10.2 硬停止或强限制节点

- 制剂学改变较大，但没有任何支持局部递送或系统暴露判断的数据
- 儿童开发意图明确，但成人基础证据明显不足
- 想走激进桥接路径，但缺少可引用的同成分、同机制或同剂型依据
- 关键适应症终点尚未定义或无法操作

## 11. 质量控制机制

### 11.1 证据分级

建议优先级如下：

1. 中国 / FDA 正式指导原则
2. FDA 公开审评资料 / review package / label
3. PSG
4. 用户案例库
5. 一般研发逻辑推断

### 11.2 输出置信度

每次输出都附带：

- 总体置信度：高 / 中 / 低
- 保守路径可靠度
- 激进路径监管风险

### 11.3 失败保护

- 缺信息时继续输出，但标记不确定项
- 高风险项目自动强化风险语句
- 不输出伪精确结论

### 11.4 验证集

为 4 个优先适应症分别建立回归验证案例，用于后续升级时验证：

- 注册属性判断是否稳定
- 双路径是否稳定输出
- 桥接和系统暴露问题是否仍能识别
- 依据引用层级是否正确

## 12. 维护机制

### 12.1 静态更新

当新增以下资料时更新本地 references：

- 中国新指导原则
- FDA 新公开审评资料
- 重要新批准产品
- 用户整理的新案例库

### 12.2 动态更新

具体品种分析时，根据需要运行时抓取：

- 最新 guidance
- 最新 PSG
- 最新 review package
- 最新 label 或审批信息

### 12.3 版本化

正式输出时建议显示：

- SKILL 版本
- 资料更新时间
- 是否进行了联网核验
- 优先引用地区规则

第一版建议先实现：

1. 核心资料层
2. 输入模板
3. 决策树主干
4. 4 个适应症模块
5. 固定输出模板

暂不追求一开始就做全适应症全场景覆盖。

## 14. 总结

该 SKILL 的最佳定位是“可解释的外用制剂临床开发策略引擎”。第一版以中国和 FDA 双监管场景为框架，以改良型新药规则为重心，以 4 个优先适应症模块为深度支点，并通过保守/激进双路径输出支持不同开发偏好。

从设计上看，该项目适合采用“静态规则库 + 审评案例层 + 运行时官方检索”的混合模式，而不适合做成单一长提示词。后续如果进入实现阶段，应优先保证规则树、依据卡片和输出模板的稳定性，再逐步扩展更多适应症和案例。
