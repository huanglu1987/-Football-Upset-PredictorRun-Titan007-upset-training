# 球探冷胜预测模型设计

## 1. 背景与目标

目标是基于可稳定访问的足球赔率数据源，建立一个可复用的冷胜预测工作流，并最终封装为两个独立入口：

- 训练技能：回填历史数据、训练模型、生成回测结果
- 预测技能：输入比赛日期范围，输出逐场冷胜概率和每日排行榜

首版只聚焦 `1X2` 冷胜，不依赖球队基础面。

## 2. 已确认的首版边界

### 2.1 数据范围

- 数据来源：优先选择结构稳定、可自动化访问的数据源
- 联赛范围：五大联赛
- 时间范围：近 3 个赛季
- 特征范围：只用 `1X2`、亚盘、大小球的公开赔率信息
- 预测时点：临场版，以开赛前最后一次可抓到的赔率为准

当前默认的数据源策略为：

- 历史回测基线：`football-data.co.uk`
- 自动化抓取主方案：`API-Football`
- 如果后续能稳定拿到其他官方或商业赔率 API，可再接入统一采集层

### 2.2 目标定义

首版仅预测两类结果：

- `home_upset_win`：主队最终赢球，且赛前不是 `1X2` 最低赔一方
- `away_upset_win`：客队最终赢球，且赛前不是 `1X2` 最低赔一方

其余比赛统一归为：

- `non_upset`

平局不纳入首版主任务，后续可作为第二个独立标签扩展。

### 2.3 输出形式

预测技能需要同时输出：

- 逐场概率：每场比赛的主冷胜概率、客冷胜概率
- 每日排行榜：按冷胜概率排序的当日 Top N 候选
- 解释信息：每场附带简短赔率解释，避免黑盒输出

## 3. 设计方案选择

采用“结构化赔率特征 + 主模型”的方案：

- 先把球探公开页的赔率信息整理成统一快照
- 再生成纯赔率特征
- 最后基于时间切分做多分类冷胜模型

不采用以下路线作为首版主方案：

- 纯规则系统：可做基线，但不作为主交付
- 两阶段筛选模型：复杂度过高，首版容易过度设计

## 4. 系统模块

首版系统拆成 4 个模块：

### 4.1 采集层

负责抓取：

- 历史赔率数据文件或 API 响应
- 比赛详情标识
- `1X2` 赔率数据
- 亚盘数据
- 大小球数据

采集层需要保留原始响应缓存，便于后续重跑、排错和数据源切换排查。

### 4.2 标准化层

负责把不同页面整理成统一结构：

- 比赛唯一标识
- 联赛、赛季、开赛时间、主客队、比赛结果
- `1X2`、亚盘、大小球的初赔与临场值
- 可选的盘口变化补充字段

### 4.3 特征层

负责从标准化数据生成纯赔率特征，不引入球队积分、排名、近况等基础面信息。

### 4.4 建模与输出层

负责：

- 训练冷胜模型
- 做时间切分回测
- 生成逐场概率
- 生成每日 Top N 排行榜
- 输出简短赔率解释

## 5. 数据结构设计

首版建议保留三层数据，而不是只保留最终训练表。

### 5.1 `raw_matches`

每场一行，存放：

- 球探比赛 ID
- 联赛
- 赛季
- 开赛时间
- 主队、客队
- 最终比分
- 最终赛果

### 5.2 `raw_odds_snapshots`

每场一行或按盘口类型拆分，存放：

- `1X2` 初赔与临场
- 亚盘初盘、临场盘、水位变化
- 大小球初盘、临场盘、水位变化
- 若页面可得，则附加：
  - 变化次数
  - 最大/最小值
  - 最后变动时间

### 5.3 `model_features`

每场比赛一行，存放：

- 训练特征
- 标签
- 数据完整度标记
- 回测切分标记

## 6. 标签设计

首版主模型采用 3 分类：

- `home_upset_win`
- `away_upset_win`
- `non_upset`

这样可以直接输出：

- 主冷胜概率
- 客冷胜概率
- 非冷胜概率

相较于两个独立二分类模型，3 分类更适合互斥结果建模，也更方便做概率校准和排行榜输出。

## 7. 特征设计

首版只保留纯赔率特征，分为四组。

### 7.1 `1X2` 静态特征

- 主/平/客初赔
- 主/平/客临场赔率
- 临场最低赔一方
- 主客赔率比值
- 主赔与客赔差距
- 平赔相对位置

### 7.2 `1X2` 变化特征

- 主/平/客从初赔到临场的绝对变化
- 主/平/客从初赔到临场的相对变化率
- 上调/下调方向标记
- 主客两边是否背离

### 7.3 亚盘联动特征

- 初盘盘口位置
- 临场盘口位置
- 初盘水位与临场水位
- 升盘/降盘标记
- 热门方向是否强化
- 亚盘变化与 `1X2` 是否一致

### 7.4 大小球联动特征

- 初始大小球盘口
- 临场大小球盘口
- 水位变化
- 升盘/降盘标记
- 与 `1X2` 是否形成背离或共振

## 8. 关键约束与降级策略

不同数据源的历史深度和盘口粒度可能不同，因此首版必须显式支持降级：

- 如果拿不到完整变化明细，则只使用 `初赔` 与 `临场` 的双时点特征
- 如果缺少 `1X2` 初赔或临场，则该场不进入训练集
- 如果缺少亚盘或大小球，则允许保留该场，并使用缺失特征策略
- 如果预测日某场数据缺失，则输出结果时附带数据完整度标记

首版不假设一定能得到完整盘口时间序列，也不建设实时轮询系统。

## 8.1 首版数据源选择原则

首版按以下优先级选源：

1. 能稳定自动化访问
2. 能覆盖五大联赛近 3 赛季
3. 能提供至少 `1X2 + 大小球`，最好同时提供亚盘
4. 能区分开盘与临场，或提供可替代的历史快照

在这个原则下：

- `football-data.co.uk` 适合作为历史训练与回测基线
- `API-Football` 更适合作为后续自动化预测入口的数据源

## 9. 抓取策略

抓取流程建议为：

1. 按联赛和赛季获取比赛列表或历史文件
2. 提取比赛 ID、开赛时间、对阵信息
3. 获取 `1X2`、亚盘、大小球数据
4. 解析初赔、临场及可用的变化字段
5. 缓存原始响应和标准化结果

采集层需要支持：

- 重试
- 限速
- 字段缺失日志
- 页面结构变化告警

## 10. 模型与回测

### 10.1 首版模型

首版优先采用可解释、训练稳定的树模型，例如：

- LightGBM
- XGBoost

同时保留一版简单规则或逻辑回归作为基线。

### 10.2 回测方式

必须使用时间切分，不允许随机切分。

可选方案：

- 前两赛季训练，后一赛季验证
- 滚动窗口回测

### 10.3 评估指标

首版优先关注：

- `home_upset_win` / `away_upset_win` 的 precision、recall
- Top N 命中率
- 概率校准表现
- 各联赛稳定性

不以整体准确率作为唯一目标。

## 11. 双技能设计

### 11.1 训练技能

输入建议包括：

- 联赛范围
- 赛季范围
- 是否强制重抓
- 是否强制重训

训练技能负责：

- 历史数据回填
- 特征生成
- 时间切分回测
- 模型文件落盘
- 回测报告输出

### 11.2 预测技能

输入建议包括：

- `start_date`
- `end_date`
- 可选 `league`
- 可选 `top_n`

预测技能负责：

- 抓取日期区间内的比赛赔率快照
- 载入已训练模型
- 输出逐场冷胜概率
- 输出每日排行榜
- 输出每场的赔率解释

## 12. 首版不做的内容

为了保持小步快跑，首版明确不做：

- 平局冷门模型
- 多来源赔率融合
- 实时盘口轮询
- 自动重训调度
- 球队基础面特征
- 全量盘口时间序列数据库

## 13. 验收标准

首版完成后，需要满足以下标准：

- 能稳定回填五大联赛近 3 赛季目标字段
- 能完成至少一套严格时间切分回测
- 能按日期范围输出逐场冷胜概率与每日 Top N
- 每场结果附带可读的赔率解释
- 能在数据不完整时进行降级，而不是静默失败
