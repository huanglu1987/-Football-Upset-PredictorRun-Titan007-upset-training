# Case Library Index

## Purpose

This case library is the teammate-friendly navigation layer above the raw worked examples.

Use it when:

- a colleague downloads the repo and needs to find the closest precedent quickly
- the current project is easier to classify by disease family or mechanism family
- the team wants reusable development patterns rather than one-off example reading

## How To Use

1. First choose one axis:
   - `按适应症`
   - `按机制/开发类型`
2. Read the family card.
3. Then jump from the family card into the closest worked example.
4. Reuse the logic, not the exact conclusion.

The worked examples remain the concrete demonstration layer.

The case-library files answer:

- which example should I start from
- what development pattern am I really dealing with
- what common failure modes should I watch first
