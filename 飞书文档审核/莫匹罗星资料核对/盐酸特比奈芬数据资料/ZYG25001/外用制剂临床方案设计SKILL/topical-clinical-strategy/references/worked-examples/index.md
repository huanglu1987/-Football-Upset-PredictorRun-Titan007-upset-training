# Worked Examples Index

## Purpose

Use these worked examples to understand what a strong strategy answer should look like in representative real-world scenarios.

These are not binding templates. They are demonstration-grade examples to help:

- team trial runs
- output-style alignment
- manual QA
- onboarding colleagues who are new to the skill

## How to Use

1. Pick the example closest to the current user request.
2. Compare the user input with the example input framing.
3. Reuse the output shape, not the exact conclusion.
4. Keep the current answer grounded in the actual evidence for the current product.

## Rule

If the current case materially differs from the worked example, say so explicitly instead of stretching the example too far.
